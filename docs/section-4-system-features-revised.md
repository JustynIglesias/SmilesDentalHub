## 4. System Features

This section presents the major features and functionalities of the proposed Smiles Dental Hub web system. It explains how the system responds to user actions and how each authorized user role is supported in performing clinic-related tasks efficiently and accurately.

### 4.1 Stimulus/Response

This subsection presents the stimulus-and-response interactions between the users and the web system. It describes the actions initiated by users and the corresponding responses generated by the system, showing how the system processes requests and provides appropriate outputs.

#### Table 4.1.1 Stimulus/Response

| Stimulus from the User | Response from the Web System |
| --- | --- |
| **Patient** |  |
| 1. The clinic staff opens the **Add Patient Record** page and provides the clinic-owned, internet-connected device to the patient. | The system displays the patient registration form. |
| 2. The patient fills in the required patient information and clicks **Next**. | The system validates the entered data. If a duplicate patient record already exists, the system displays a notification. Otherwise, it proceeds to the next set of forms. |
| 3. The patient completes the remaining forms and clicks **Next**. | The system validates the submitted information and redirects the user to the authorization and release consent section. |
| 4. The patient agrees to the authorization and release consent and submits the form. | The system enables the **Submit** button, saves the patient record, and displays a confirmation message indicating successful submission. |
| **Receptionist** |  |
| 1. The receptionist enters valid credentials and clicks **Login**. | The system verifies the credentials. If valid, the receptionist is redirected to the **Home** page; otherwise, an error message is displayed. |
| 1.1. The receptionist clicks **Forgot Password** after failing to log in or forgetting the current password. | The system requests the registered username or email, sends a verification code to the registered email address, validates the submitted code, and allows the receptionist to set a new password. |
| 2. After logging in, the receptionist accesses the available modules: **Home**, **Patient Records**, **Add Patient Record**, **Procedures**, **Patient Logs**, **My Account**, and **Log Out**. | The system displays the dashboard and navigation panel based on the receptionist's role permissions. |
| 2.1. The receptionist selects **Home**. | The system displays the dashboard with clinic summary information, including the number of catered patients. |
| 2.2. The receptionist selects **Patient Records**. | The system displays the patient records list, where the receptionist can search, view, and manage patient information based on permitted actions. |
| 2.2.1. The receptionist clicks **View** beside a patient record. | The system opens the detailed patient record with sub-tabs such as **Patient Information**, **Dental Records**, and **Service Records**. |
| 2.2.1.1. The receptionist selects **Patient Information**. | The system displays the patient's personal details. The receptionist can view, update, upload supporting documents, and export the patient record as a PDF. |
| 2.2.1.2. The receptionist selects **Dental Records**. | The system displays the patient's dental records. The receptionist can view the information but cannot modify the dental record contents. |
| 2.2.1.3. The receptionist selects **Service Records**. | The system displays the patient's service history, where the receptionist can view, add, and update service records. |
| 2.2.2. The receptionist changes the status switch beside a patient record. | The system updates the patient record status to **Active** or **Inactive**. Inactive records are transferred to the **Inactive List**. |
| 2.3. The receptionist selects **Add Patient Record**. | The system displays the patient registration form. |
| 2.4. The receptionist selects **Procedures**. | The system displays the procedures module with tabs for **Services** and **Dental Chart Legends**. |
| 2.4.1. The receptionist selects **Services**. | The system displays the list of dental services offered by the clinic, where the receptionist can view, add, and update service entries according to access permissions. |
| 2.4.2. The receptionist selects **Dental Chart Legends**. | The system displays the list of tooth conditions and chart legends available in the system, where the receptionist can view and update legend information according to access permissions. |
| 2.5. The receptionist selects **Patient Logs**. | The system displays the patient activity logs. |
| 2.6. The receptionist selects **My Account**. | The system displays the My Account page, where the user can update profile details such as birthday, mobile number, and address, and may request an email change that requires verification through a code sent to the new email address. |
| 2.6.1. The receptionist logs in using a newly created staff account with an incomplete placeholder profile. | The system requires the receptionist to complete the missing personal details, submit a real email address, and verify the email through a one-time code before full access is granted. |
| 2.7. The receptionist selects **Log Out**. | The system ends the current session and redirects the user to the **Login** page. |
| **Associate Dentist** |  |
| 1. The associate dentist enters valid credentials and clicks **Login**. | The system verifies the credentials. If valid, the associate dentist is redirected to the **Home** page; otherwise, an error message is displayed. |
| 1.1. The associate dentist clicks **Forgot Password**. | The system sends a password recovery verification code to the registered email address, validates the submitted code, and allows the associate dentist to create a new password. |
| 2. After logging in, the associate dentist accesses the available modules: **Home**, **Patient Records**, **Add Patient Record**, **Procedures**, **Patient Logs**, **My Account**, and **Log Out**. | The system displays the dashboard and navigation panel based on the associate dentist's role permissions. |
| 2.0. The associate dentist selects **Home**. | The system displays the dashboard with current patient statistics and summary charts. |
| 2.1. The associate dentist selects **Patient Records**. | The system displays the patient records list and allows access to detailed patient information. |
| 2.1.1. The associate dentist clicks **View** beside a patient record. | The system opens the detailed patient record with **Patient Information**, **Dental Records**, and **Service Records** tabs. |
| 2.1.1.0. The associate dentist selects **Patient Information**. | The system displays the patient's personal, medical, and dental history details and allows permitted updates, document viewing, document uploads, and PDF export. |
| 2.1.1.1. The associate dentist selects **Dental Records**. | The system displays the patient's dental records and allows the associate dentist to update dental chart information, notes, prescriptions, and related dental details. |
| 2.1.1.2. The associate dentist selects **Service Records**. | The system displays all service records and allows the associate dentist to view, add, and update service information when needed. |
| 2.2. The associate dentist selects **Procedures**. | The system displays the services list and dental chart legends used in patient treatment documentation. |
| 2.3. The associate dentist selects **My Account**. | The system displays the My Account page and allows the associate dentist to update profile details and submit an email-change request subject to verification. |
| 2.4. The associate dentist selects **Log Out**. | The system ends the session and redirects the user to the **Login** page. |
| **Admin** |  |
| 1. The admin enters valid credentials and clicks **Login**. | The system verifies the credentials. If valid, the admin is redirected to the **Home** page; otherwise, an error message is displayed. |
| 1.1. The admin clicks **Forgot Password**. | The system sends a verification code to the registered email address, validates the submitted code, and allows the admin to reset the password securely. |
| 2. After logging in, the admin can access **Home**, **Patient Records**, **Add Patient Record**, **Procedures**, **Patient Logs**, **My Account**, **Admin**, and **Log Out**. | The system displays the dashboard and complete navigation menu for administrative functions. |
| 2.0. The admin selects **Patient Records** and opens a patient record. | The system displays the detailed patient information, dental records, and service records, and allows the admin to perform all permitted record, document, and export actions. |
| 2.1. The admin selects **Procedures**. | The system displays the procedures module with tabs for **Services** and **Dental Chart Legends**. |
| 2.1.1. The admin selects **Services**. | The system displays the list of services, where the admin can view, add, update, archive, and retrieve service records when applicable. |
| 2.1.2. The admin selects **Dental Chart Legends**. | The system displays the list of tooth conditions and chart legends, where the admin can view, add, update, archive, and retrieve entries. |
| 2.2. The admin selects the **Admin** tab. | The system opens the admin page with sub-tabs such as **Manage Users**, **Inactive List**, and **Archive List**. |
| 2.2.1. The admin selects **Manage Users**. | The system displays all registered active staff users and their assigned roles. The admin can add, view, update, archive, and manage user accounts. |
| 2.2.1.1. The admin adds a new user account. | The system validates the submitted account information, creates the staff account, generates a system email format when applicable, and displays a success or error message. |
| 2.2.1.2. The admin updates an existing user account. | The system validates the revised details and saves the allowed profile, role, and account changes. |
| 2.2.2. The admin selects **Inactive List**. | The system displays inactive patient records that may be archived by the admin. |
| 2.2.3. The admin selects **Archive List**. | The system displays archived records. Depending on the selected archive type, the admin may retrieve archived patients, users, services, or dental chart legends. |
| 2.3. The admin selects **My Account**. | The system displays the My Account page for updating the admin's own profile information and handling verified email-change requests. |
| 2.4. The admin selects **Log Out**. | The system ends the session and redirects the user to the **Login** page. |

Table 4.1.1 presents the interaction between the users and the web-based system. It shows how the system responds to the actions performed by different users, namely the patient, receptionist, associate dentist, and admin. The table demonstrates how the system handles major functions such as patient registration, login validation, record viewing, record updating, service management, and user administration. Through these interactions, the system is able to support clinic operations accurately, efficiently, and securely.
